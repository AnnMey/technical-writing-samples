User roles layout proposal

Original input

The system shall authorize the Operator role to perform the following tasks:

- Requests access

- Results access

- Cancel orders

- Validate results

The system shall authorize the Operator plus role to perform the following tasks:

- All "Operator" role tasks

- Reject results

- System settings access

The system shall authorize the Supervisor role to perform the following tasks:

- All "Operator Plus" role tasks

- User management access

Edited content

The role assigned to a user determines what the user can do in the system. Each user can have one of the following roles: Operator, Operator plus, or Supervisor.

Users with the Operator role have the following permissions:

  • Accessing requests

  • Accessing results

  • Canceling orders

  • Validating results

Users with the Operator plus role have the following permissions:

  • All the tasks covered by the Operator role

  • Rejecting results

  • Accessing system settings

Users with the Supervisor role have the following permissions:

  • All the tasks covered by the Operator plus role

  • Accessing user management
