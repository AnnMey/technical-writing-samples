The role assigned to a user determines what the user can do in the system. Each user can have one of the following roles: Operator, Operator plus, or Supervisor.

Users with the Operator role have the following permissions:

  • Accessing requests

  • Accessing results

  • Canceling orders

  • Validating results

Users with the Operator plus role have the following permissions:

  • All the tasks covered by the Operator role

  • Rejecting results

  • Accessing system settings

Users with the Supervisor role have the following permissions:

  • All the tasks covered by the Operator plus role

  • Accessing user management
